Maven基础知识

2019-08-19

Maven

1. Maven的作用

  • 增加第三方Jar (commons-fileupload.jar commons-io.jar)
  • jar包之间的依赖关系 （commons-fileupload.jar 自动关联下载所有依赖的Jar，并且不会冲突）

2. Maven的概念

  • 是一个基于Java平台的 自动化构建工具
  • 总的来说：自动化构建工具maven：将 原材料（java、js、css、html、图片）->产品（可发布项目）

2.1 管理过程

  • 清理：删除编译的结果，为重新编译做准备。
  • 编译：java->class
  • 测试： 针对于 项目中的关键点进行测试，亦可用 项目中的测试代码 去测试开发代码；
  • 报告：将测试的结果 进行显示
  • 打包： 将项目中包含的多个文件 压缩成一个文件， 用于安装或部署。 （java项目-jar、web项目-war）
  • 安装：将打成的包 放到 本地仓库，供其他项目使用。
  • 部署：将打成的包 放到 服务器上准备运行。

3. 配置Maven

  • 第一步：配置JAVA_HOME

  • 第二步：配置MAVEN_HOME或者M2_HOME

    • D:\apache-maven-3.5.3\bin

  • 第三步：配置path

    • %MAVEN_HOME%\bin

  • 第四步：验证，cmd中输入

    • mvn -version
    • 出现版本号表示配置成功

  • 第五步：配置本地仓库

    • 打开：maven目录/conf/settings.xml

    • 默认本地仓库 ：C:/Users/YANQUN/.m2/repository

    • 修改为： D:/mvnrep

4. Maven的约定

5. 通过Maven创建Java项目

  • 前提：配置maven

  • 第一步：new —-> Maven Project —->

6. 运行Maven项目

6.1 如何运行：

  • 右键项目 —-> run as —-> Maven build… —->

6.2 Maven命令

  • compile 编译项目的源代码。
  • test 使用合适的单元测试框架运行测试。这些测试代码不会被打包或部署。
  • package 接受编译好的代码，打包成可发布的格式，如JAR。
  • install 将包安装至本地仓库，已让其它项目依赖。
  • clean 移除所有上一次构建生成的文件。
  • 其余还有很多命令，以上为常见命令。

7. maven生命周期

  • 3个阶段
  • 1.clean lifecycle ：清理
    • pre-clean clean post-clearn
  • 2.default lifecycle ：默认(常用)
  • 3.site lifecycle：站点
    • pre-site site post-site site-deploy

8. 依赖

8.1 依赖的范围、依赖的有效性

  • A.jar —-> B.jar

compile test provided

测试 √ √ √

  • Maven在编译、测试、运行项目时，各自使用一套classpath。
  • compile(默认)

8.2 依赖排除

8.3 依赖的传递性

  • A.jar —-> B.jar —-> C.jar
  • 要使 A.jar ->C.jar:当且仅当 B.jar 依赖于C.jar的范围是compile

8.4 依赖原则

  1. 路径最短优先原则
  2. 路径长度相同
    1. 在同一个pom.xml文件中有2个相同的依赖（覆盖）：后面声明的依赖 会覆盖前面声明的依赖（严禁使用本情况，严禁在同一个pom中声明2个版本不同的依赖）
    2. 如果是不同的 pom.xml中有2个相同的依赖（优先）：则先声明的依赖 ，会覆盖后声明的依赖

30.Date日期类(util)

31.继承（extends）

  • 继承父类
  • 目的：减少冗余（重复）
  • 如果很多类的属性、方法 重复出现，重复 可以提取到父类。
  • 注意：如果方法名相同、方法体不同，则不能提取到父类。
  • super关键字：跨过子类，直接去父类中调用。
  • this关键字：先在当前类中找，没有就去父类。
  • private修饰的、构造方法 是不能被继承的。但是 子类可以显示的调用父类构造super

32.构造方法

  • 如果类中 没有任何构造方法，则系统自动提供一个无参构造，如果有则不会自动提供，需要手写。

  • 一般建议：如果给类中 编写构造方法，则手动编写一个无参构造 防止报错

  • this()调用本类构造

  • super()调用父类构造

  • 如果在构造方法中 通过this(..)/super()调用其他的构造方法，则 this()/super()必须放在第一行。

33.静态代码块

  • 静态代码块: 第一次执行。永远只执行一次 。初始化（人机猜拳 ：人 计算）

  • 普通代码块: 在每次执行无参构造之前执行。

34.方法重载、方法重写

  • 方法重载：方法名相同，参数列表不同，与返回值无关
  • 方法重写：子类对父类方法的重写，方法名，参数列表相同。
    • 返回值也相同或是子类。
    • 访问修饰符不能比父类小。
    • 构造方法不能被重写，因为不能被继承。

35.final关键字

  • final修饰的类 ：不能被继承
  • final修饰的方法：不能被重写
  • final修饰的变量、属性：不能被修改

36.抽象类（abstract ）

  • 抽象方法：abstract ，没有方法体

  • 抽象方法必须包含在抽象类中

  • 抽象类中，并不一定包含抽象方法

  • 抽象类不能实例化（new X()），原因：抽象类中可能存在抽象方法，而抽象方法没有方法体

  • 子类必须重写父类中的所有抽象方法

37.多态

  • 简单来说：歧义 ，一个词语 必须根据上下文才有实际的含义

  • 父类 引用对象名字 = new 子类()；

    • 就是说父类的引用指向子类的对象。

  • 一般步骤：

    • 写一个父类
    • 子类重写父类的方法
    • 运行时：父类 引用对象名字 = new 子类()；

  • 父类和子类之间的转换

  • 如何判断 该对象是 该类型

    • 对象 instanceof 类型

  • 4.使用父类作为方法的返回值

39.接口（interface）

  • 接口比抽象类 更进一步的“抽象”

  • 接口中的所有方法默认public abstract

  • 接口中的属性默认static final

  • 接口中不存在构造方法

40.实现（implements）

  • 实现接口
  • 实现类必须实现接口中的 抽象方法

  • 注意：

    • dom对象 只适用于js的各种语法（函数、属性），jquery对象只用于jquery的各种语法（函数、属性）。

    • dom对象和jquery对象的各自独立。

  • 建议：

    • js对象 直接写title
    • jquery 加上$，例如$title
